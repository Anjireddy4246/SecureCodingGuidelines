# Fundamentals
The following general principles apply throughout Java security.

- FUNDAMENTALS 0-0 [Prefer to have obviously no flaws rather than no obvious flaws](g00)
- FUNDAMENTALS 0-1 [Design APIs to avoid security concerns](g01)
- FUNDAMENTALS 0-2 [Avoid duplication](g02)
- FUNDAMENTALS 0-3 [Restrict privileges](g03)
- FUNDAMENTALS 0-4 [Establish trust boundaries](g04)
- FUNDAMENTALS 0-5 [Minimise the number of permission checks](g05)
- FUNDAMENTALS 0-6 [Encapsulate](g06)
- FUNDAMENTALS 0-7 [Document security-related information](g07)
